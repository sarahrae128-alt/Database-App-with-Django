## Mission 3: Connect Django to Supabase

**Prompt:** Help me configure this Django project to connect to PostgreSQL using a DATABASE_URL environment variable stored as a GitHub Codespaces secret. Identify any needed Python packages, show the install command, update requirements.txt, and update settings.py without hard-coding the database URL or password in any repository file.

**Result:** AI suggested installing `psycopg2-binary` and `dj-database-url`, and showed how to update `settings.py` to use `os.environ.get('DATABASE_URL')` with `dj_database_url.parse()`. It also showed how to add the secret in GitHub Codespaces settings.

**My changes:** Verified the packages were not already installed, confirmed the DATABASE_URL was stored as a Codespace secret (not in any file), and tested the connection by running `python manage.py migrate`.

**Verification:** Ran `python manage.py migrate` successfully, then visited `/admin/` and confirmed the login page loaded.

## Mission 4: Generate Models with inspectdb

**Prompt:** Explain these Django models generated by inspectdb. Which models represent existing database tables, and why did Django mark them as unmanaged?

**Result:** AI explained that `managed = False` tells Django not to create, alter, or delete these tables during migrations, because they already exist in Supabase. Each class maps to an existing table via `db_table` in the `Meta` class.

**My changes:** Reviewed each model class and compared field names to the Supabase table editor to confirm they matched. Did not change `managed = False` on any existing model.

**Verification:** Ran `python manage.py check` with no errors, and confirmed models appeared in the Django shell.
